## Ход выполнения в Ubuntu Desktop (команды)
### 1. Подготовка окружения
```bash
sudo apt update
sudo apt install --no-install-recommends git cmake ninja-build gperf \
  ccache dfu-util device-tree-compiler wget python3-dev python3-venv python3-tk \
  xz-utils file make gcc gcc-multilib g++-multilib libsdl2-dev libmagic1
```

Для ARM64 (если `gcc-multilib` не ставится) запускаем без `gcc-multilib g++-multilib`.

### 2. Виртуальное окружение и исходники Zephyr
```bash
python3 -m venv ~/zephyrproject/.venv
source ~/zephyrproject/.venv/bin/activate
pip install west

west init ~/zephyrproject
cd ~/zephyrproject
west update
west zephyr-export
west packages pip --install
```

### 3. SDK и сетевые утилиты
```bash
cd ~/zephyrproject/zephyr
west sdk install

sudo apt install -y wireshark tcpdump netcat-openbsd
```

### 4. Сборка и запуск echo_server
```bash
west build -b native_sim samples/net/sockets/echo_server -- -DEXTRA_CONF_FILE=overlay-nsos.conf
west build -t run
```

### 5. Перехват и отправка данных (3 терминала)
Терминал 1: сервер уже запущен (`west build -t run`).

Терминал 2 (поиск порта и перехват):
```bash
ss -tnlp | grep zephyr
sudo tcpdump -A -i lo port <PORT>
```

Терминал 3 (клиент):
```bash
nc 127.0.0.1 <PORT>
```

Терминал 3, пример вводимых строк:
```text

I love Zephyr. Zephyr and ASSM forever
```

Ожидаемый результат: введенные строки в явном виде видны в `tcpdump` (терминал 2).

## Ответы на контрольные вопросы
1. Какие функции безопасности есть в Zephyr?
- Разделение привилегий `user mode`/`supervisor mode`.
- Ограничение доступа к памяти через MPU/MMU и memory domains.
- Контроль доступа к kernel objects и драйверам, в том числе проверка системных вызовов.
- Механизмы защиты стека (stack sentinel, hardware stack protection, stack canaries).
- Рекомендации hardening через встроенный инструмент `hardenconfig`.

2. Используют ли приложения одно и то же адресное пространство с ядром?
- Зависит от платформы и конфигурации.
- На платформах без MMU/MPU Zephyr часто работает как монолит: код приложения и ядра в одном общем адресном пространстве.
- При включенном userspace + MPU/MMU доступ разделяется и ограничивается.

3. Есть ли у Zephyr неисполняемый стек и ASLR?
- Полноценный ASLR уровня «как в Linux userspace» в Zephyr обычно не применяется как базовый механизм для большинства MCU-сценариев.
- Есть защита стека и связанные hardening-опции; также есть архитектурные механизмы, зависящие от конкретного CPU/MPU/MMU.
- Итог: наличие/эффективность NX-подобных ограничений и рандомизации зависит от архитектуры и Kconfig.

4. Работают ли в Zephyr стандартные атаки (например, переполнение буфера)?
- Да, если отключены/неподдерживаются механизмы memory protection, переполнение может приводить к повреждению памяти.
- При userspace + MPU/MMU + stack protections простые переполнения чаще приводят к fault/остановке потока вместо тихого захвата управления.
- Но логические уязвимости и ошибки конфигурации остаются возможны.

5. Как понять, передан пустой пакет или пакет с сообщением?
- В `tcpdump`/Wireshark у пакета с данными виден payload (текст после заголовков) и ненулевая длина полезной нагрузки.
- Для TCP фильтр в Wireshark: `tcp.len == 0` показывает сегменты без данных (ACK/служебные), а `tcp.len > 0` — сегменты с сообщением.
